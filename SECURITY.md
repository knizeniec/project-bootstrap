# Security Policy

## Reporting

- Do not open public issues for security vulnerabilities.
- Report suspected vulnerabilities privately to repository maintainers.

## Response Expectations

- Acknowledge report receipt promptly.
- Triage severity and scope before remediation.
- Document remediation and verification in repository records.

## Baseline

- Follow secure-by-default engineering practices.
- Map high-risk controls to OWASP ASVS when relevant.
